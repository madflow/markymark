# AGENTS.md

Guidelines for agentic coding agents working in this repository.

## Project Overview

`markymark` is a Go CLI tool that renders a Markdown file as HTML and serves it locally in a browser at `http://localhost:3000`. It supports a `--watch` / `-w` flag that live-reloads via Server-Sent Events (SSE) on file changes. It auto-detects a `README.md` in the current directory when no file argument is given.

- **Module:** `github.com/madflow/markymark`
- **Go version:** 1.26+
- **Entry point:** `main.go` (package `main`)

## Build, Generate, and Run Commands

```bash
# Generate templ files then compile the binary
make build

# Only regenerate *_templ.go files from *.templ sources
make generate          # or: templ generate

# Run (build first, then execute)
make run

# Watch mode for templ regeneration during development
make watch             # or: templ generate -w

# Remove binary and all generated *_templ.go files
make clean
```

> **Important:** Always run `make generate` (or `make build`) after editing any `.templ` file.
> Never manually edit `*_templ.go` files — they carry the header `// Code generated by templ - DO NOT EDIT.`

## Lint and Format Commands

```bash
# Run all linters (golangci-lint)
make check             # or: golangci-lint run

# Apply Go fix tool
make fix               # or: go fix ./...

# Format code with gofumpt (run via golangci-lint or directly)
gofumpt -w .
```

Linter configuration is in `.golangci.yml`. Enabled linters:
- `bodyclose` — ensures HTTP response bodies are closed
- `gocritic` — opinionated code style checks
- `misspell` — spell-check in comments and strings

Formatter: `gofumpt` (stricter variant of `gofmt`).

## Code Style Guidelines

### Formatting

- Use `gofumpt` (not plain `gofmt`) for formatting. Run `make check` to validate.
- Generated files are excluded from lint/format checks (`generated: lax` in `.golangci.yml`).

### Imports

Group imports in two blocks — stdlib first, then external — separated by a blank line:

```go
import (
    "bytes"
    "context"
    "log"
    "net/http"

    "github.com/a-h/templ"
    "github.com/fsnotify/fsnotify"
    "github.com/madflow/markymark/internal/markdown"
)
```

Do not mix stdlib and external packages in the same block.

### Naming Conventions

| Construct | Convention | Example |
|-----------|-----------|---------|
| Packages | lowercase single word | `finder`, `markdown`, `server` |
| Exported types/funcs | `PascalCase` | `Server`, `FindReadme()`, `TemplatePage()` |
| Unexported types/funcs | `camelCase` | `resolveFilePath()`, `cssReset()` |
| Struct fields (unexported) | `camelCase` | `baseDir`, `watchMode`, `sseClients` |
| Variables | `camelCase` | `absPath`, `watchMode` |
| Templ components (exported) | `PascalCase` | `TemplatePage` |
| Templ components (unexported) | `camelCase` | `headerComponent`, `reloadScript` |

### Error Handling

Use idiomatic Go error handling — no panics for recoverable errors.

1. **Fatal errors at startup** — use `log.Fatal` / `log.Fatalf` when the program cannot continue:
   ```go
   absPath, err := filepath.Abs(filePath)
   if err != nil {
       log.Fatal(err)
   }
   ```

2. **Logged non-fatal errors** — use `log.Printf` for runtime errors where execution can continue:
   ```go
   if err := browser.OpenURL("http://" + addr); err != nil {
       log.Printf("open browser: %v", err)
   }
   ```

3. **Silent fallback** — return a zero/empty value when the error is benign and callers handle absence:
   ```go
   entries, err := os.ReadDir(".")
   if err != nil {
       return ""
   }
   ```

4. **Intentional ignore** — use `_` only when an error is genuinely unrecoverable and already handled:
   ```go
   _ = watcher.Add(s.filePath)
   ```

5. **Error message format:** `"<operation>: <detail>: %v"` — e.g. `"watch: create watcher: %v"`, `"read %s: %v"`.

### Types

- Prefer explicit types over `interface{}` / `any` unless genuinely polymorphic.
- Use type aliases and function types (`type componentFunc func(...)`) for clarity when passing functions as values.
- Struct fields should be unexported unless they need to be accessed from another package.

## Commit Message Convention

This repository uses **Conventional Commits**:

```
<type>(<scope>): <short summary>

[optional body]

[optional footer(s)]
```

Common types: `feat`, `fix`, `refactor`, `test`, `docs`, `chore`, `ci`, `build`.

Examples:
```
feat(server): add configurable port flag
fix(markdown): handle images with relative paths correctly
chore(deps): update fsnotify to v1.9.0
docs: add usage examples to README
test(finder): add unit tests for case-insensitive detection
```

- Use the imperative mood in the summary (`add`, `fix`, `update` — not `added`, `fixed`).
- Keep the summary under 72 characters.
- Reference issues in the footer: `Closes #42`.

## Key Implementation Details

- The HTTP server always binds to `localhost:3000` (hardcoded in `main.go`).
- A **50ms debounce** is applied to filesystem events before re-rendering in watch mode.
- The watcher handles editors that rename-replace files on save (vim, IntelliJ) by re-adding the watch after a rename event.
- `CGO_ENABLED=0` is used for release builds (fully static binary, no C dependencies).
- The `.gitignore` excludes the compiled `markymark` binary — never commit it.
